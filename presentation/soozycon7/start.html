What is Angelos?

  • PerlのWAFの拡張性の高さ（何でもプラガブル）を活かしつつ
  • Django, Railsのようなフルスタックを実現できる
  • WSGIに対応した
  • MousefulなWAF

Angelosの設計の基本方針

  • コアを小さく
    • Config, Logger, I18N, Pluginの分離
  • 適切な箇所を拡張できる
    • Controller, View, Middleware
  • 適切なデフォルトセットを提供
    • Plugin, Middleware
  • デフォルトの規約を設ける
    • Routing, Project Structure, クラス構成
  • 良いアーキテクチャに自然に(MMVC, Service Classの共有)

AngelosのWAFの基本部分

  • Dispatcher
    • RESTful dispatcher - HTTP::Router
  • Component Loader
    • DI Container - Bread::Board(?)
  • WSGI
    • HTTP::Engine, HTTP::Engine::Middleware

AngelosのWAF周辺機能

  • プラグイン系
    • Mouse + MouseX::Object::Pluggable (modifier + α)
  • Scaffold + Application Templates
    • Module::Setup
  • CLI
    • App::Cmd
